SECTION C: LIVE FIELD VIEW (MATCH ANIMATION)
================================================================================

Purpose: Shows realistic cricket match animation in real-time

CRICKET FIELD OVAL CONTAINER:

Base Field:
- Green field background with layered grass texture
- Realistic shading with radial gradients
- Multiple light/shadow layers for depth
- Subtle grass pattern (repeating subtle radial gradient)
- Drop shadow effect beneath oval

Inner Structures:

Dashed Inner Circle:
- 30-yard ring boundary
- White dashed line
- Semi-transparent
- Positioned at ~18% inset from field edges

Brown Pitch (Center Strip):
- Central brown/tan rectangular area
- Gradient shading (darker top, lighter bottom)
- White border outline
- Width: 32px, Height: 60% of field height
- Represents the cricket wicket/pitch

White Creases:
- Two horizontal white lines
- One at top (bowler end)
- One at bottom (batter end)
- Represent creases where wickets are placed

Wicket/Stumps:
- Small white horizontal rectangles at each crease
- Height: 5px, Width: 10px
- Represent stumps behind batsman and at far end

Animated Elements:

WHITE BALL:
- Circular shape (11px diameter)
- Radial gradient shading (lighter top-left, darker bottom-right)
- White glow shadow effect
- Moves in realistic path:
  
  DELIVERY PHASE (0-45% of cycle):
  - Ball starts at bowler (center bottom, 49% left, 70% top)
  - Moves toward batter (center-ish, around 51.8% left, 46.5% top)
  - Arc upward in middle with easing
  - Simulates ball pitch-up trajectory
  - Uses ease-in-out-quad easing function
  
  SHOT/FLIGHT PHASE (45-100% of cycle):
  - Ball travels from pitched position to randomized shot destination
  - Destinations randomized from 6 possible targets:
    - Long-on, short cover, mid-wicket, square leg, third man area, etc.
  - Arc downward (ball falls toward ground after shot)
  - Uses ease-out-cubic easing function
  - Cycle repeats every 4.2 seconds with new random destination

BALL TRAIL:
- Thin white gradient line following ball's path
- Start fully transparent, middle semi-opaque, end transparent
- Thickness: 2.5px
- Angle rotates based on ball velocity direction
- Opacity and length change based on ball speed:
  - Faster movement = longer, more opaque trail
  - Slower movement = shorter, more transparent trail
- Blur filter for realistic motion effect
- Smooth transitions (0.08-0.15s)

FIELDER DOTS:
- 7 yellow/gold fielder positions around the field
- Each is a 13px diameter sphere
- Radial gradient fill (brighter top-left)
- Yellow glow shadow effect

Fielder Positions:
1. F1 at 16% left, 44% top (square leg area)
2. F2 at 26% left, 24% top (mid-wicket area)
3. F3 at 43% left, 14% top (mid-off area)
4. F4 at 72% right, 20% top (cover area)
5. F5 at 86% right, 44% top (point/covers area)
6. F6 at 74% right, 78% bottom (fine leg area)
7. F7 at 24% left, 80% bottom (long-on area)

Fielder Animation Behavior:

Idle Motion:
- All fielders bob gently up and down with sine wave
- Each fielder offset by 0.8 radians so they bob at slightly different times
- Creates natural "breathing" effect

Reaction to Ball:
- System calculates which fielder is nearest to ball
- Nearest fielder enters "reaction mode":
  - Scales up slightly (1.0 to 1.12× size multiplier)
  - Moves toward ball direction (up to 8px toward ball)
  - Opacity increases slightly (0.86 to 1.0)
  - Effect strength decreases with distance (max 160px reaction radius)
  
- Other fielders:
  - Continue idle motion
  - Maintain base opacity (0.86)
  - No reaction movement

All Transitions Smooth:
- Fielder position updates every frame (60fps)
- Linear interpolation for position changes
- Smooth scaling and opacity transitions

Floodlight Sweep Effect:
- Animated light sweep across field
- Gradient overlay that moves left to right
- 6.8 second cycle
- Simulates stadium floodlight sweep across field
- Opacity varies (0 → 0.45 → 0.23 → 0 over cycle)
- Creates dramatic lighting effect

Accessibility:
- Respects prefers-reduced-motion media query
- If user has motion preferences set, animations disabled

KEY INTERACTIVE BEHAVIORS
================================================================================

1. FIELD PLACEMENT INTERACTION:
- Click any position dot on oval field diagram → Position selects/deselects
- Click any chip button → Position selects/deselects
- Clicking again on selected position deselects it
- Multiple selections allowed simultaneously
- Selection counter updates with count
- Selected positions display as green tags in real-time
- Field position dots highlight green when selected
- Chips get green background when selected with checkmark visible

2. BOWLER DROPDOWN INTERACTION:
- Click dropdown to open options
- Select one bowler from list
- Selection shows in dropdown button
- Only one bowler can be selected at a time
- Dropdown closes after selection
- Updates live preview with bowler name

3. STRATEGY TEXT INTERACTION:
- Click text area to focus (cursor appears)
- Type freely to describe tactical approach
- Updates in real-time as each character entered
- Live preview shows first 55 characters of strategy
- No character limit enforced (though preview truncates)
- Placeholder disappears on focus

4. LIVE PREVIEW UPDATE:
- Watches field selections, bowler choice, strategy text
- Shows/hides based on whether any field filled
- Updates instantly when user makes any change
- Formats display with bowler, field count, strategy preview

5. FORM SUBMISSION:
- Click "Submit Tactical Decision" button
- JavaScript validates:
  - At least 1 field position selected? If no: alert message
  - Bowler selected? If no: alert message
  - Strategy text entered? If no: alert message
- If validation passes:
  - Button shows loading state with spinner
  - Sends POST request to `/decisions/submit` API
  - Payload includes: username, field positions, bowler, strategy, ball number
  - Waits for response

- On Success:
  - Button shows success message (2.5 seconds)
  - Decision Output box updates with AI feedback
  - Leaderboard refreshed with updated stats
  - User sees feedback containing score, wicket chance, runs saved

- On Error:
  - Button shows error message (2.5 seconds)
  - Alert shows error details from server
  - Decision Output box remains unchanged or shows error

- After 2.5 seconds (success or error):
  - Button reverts to normal state
  - Button re-enabled
  - User can submit another decision

6. NAVIGATION INTERACTION:
- Click any navigation pill at top
- Page smoothly scrolls to corresponding section
- Active pill highlights (visual indicator)
- As user scrolls page, corresponding nav pill auto-highlights

ANIMATION SPECIFICATIONS
================================================================================

SMOOTH TRANSITIONS:
- Standard transition: 0.25s ease
- Fast transition: 0.15s linear (ball/trail updates)
- Slow transition: 0.35s ease (form submission)

KEYFRAME ANIMATIONS:
- fieldPop: Position dot pop-in when selected (0.42s)
- chipSlideIn: Chip selection slide-in (0.36s)
- tagFadeIn: Tag appearance (0.28s)
- flicker: Stadium floodlight pulse (5s infinite)
- livePulse: Live badge pulse (1.2s infinite)
- floodlightSweep: Field lighting sweep (6.8s infinite)

EASING FUNCTIONS USED:
- Cubic-bezier(0.34,1.56,0.64,1): Bouncy/overshoot effect
- Ease-in-out-quad: Smooth ball delivery
- Ease-out-cubic: Smooth shot trajectory
- Linear: Constant pace animations

MOTION PREFERENCES:
- If user sets prefers-reduced-motion: all animations disabled
- Alternative: static content still visible, no motion


PERFORMANCE NOTES
================================================================================

Canvas Background:
- Runs requestAnimationFrame at 60fps
- Draws animated particles and field geometry
- Low performance impact due to simple shapes

Live Ground Animation:
- requestAnimationFrame loop for smooth 60fps motion
- Calculates ball position, trail, and fielder reactions each frame
- ~40 calculations per frame but minimal DOM updates
- Optimized for modern browsers

Field Position Initialization:
- Creates 14 DOM elements on page load
- Minimal initial load time
- Event listeners attached to each element
- CSS classes toggle rather than inline style updates (faster)

Leaderboard Updates:
- DOM innerHTML replaced on update
- 10 items maximum
- Uses classList.toggle for styling (efficient)

- Tab order logical (left to right, top to bottom)
- Enter/Space to activate buttons
- Enter to submit form
- Escape to close modals/dropdowns

Color Contrast:
- Text contrast ratio meets WCAG AA (4.5:1 minimum)
- Status indicators don't rely solely on color
- Additional visual indicators (icons, text) provided

Motion and Animation:
- prefers-reduced-motion respected
- Animations non-essential (content visible without motion)
- No auto-playing videos or audio
- Users can pause animations if needed

Alternative Text:
- Icons accompanied by text labels or aria-labels
- Images have alt text (though mostly decorative)
- Non-decorative graphics descriptive

Mobile Accessibility:
- Touch targets minimum 44px × 44px
- Readable text size (minimum 12px, typically 14px+)
- Forms accessible on small screens
- Zoom/magnification supported
